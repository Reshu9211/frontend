Question 1: Difference between “==” and “===” operators.

= (loose equality): Compares values after converting them to the same type if necessary.
=== (strict equality): Compares both value and type. No type conversion is done.

        Example:
        
        5 == '5'     // true (because '5' is converted to number)
        5 === '5'    // false (because number !== string)
        
        Summary:
        
        Use === to avoid unexpected results due to type coercion.

Question 2: What are the differences between var, let and const?

Var
- Global scope: If declared outside a function, var variables are global.
- Can be updated: var variables can be updated after declaration.
Let
- Cannot be accessed before declaration: let variables are in a "temporal dead zone" until declared.
- Can be updated: let variables can be updated after declaration.
Const
- Cannot be updated: const variables cannot be updated after declaration.
- Must be initialized: const variables must be initialized when declared.

Question 3: What is execution context?

An execution context is the environment in which JavaScript code is evaluated and executed. It contains the scope, variables, and the value of 'this'.

Question 4: What is creation phase and execution phase?

In the creation phase, the JavaScript engine sets up memory for variables and functions. In the execution phase, it runs the code line by line and assigns values to the variables.

Question 5: What is hoisting?

Hoisting is JavaScript's default behavior of moving declarations (variables and functions) to the top of their scope before code execution.

Question 6: Difference between undefined, not defined, and NaN

"undefined" means a variable has been declared but not assigned a value. "not defined" means the variable has not been declared at all. "NaN" stands for "Not a Number", a result of invalid number operations.

Question 7: How many operators do we have in JS?

JavaScript has many operators including arithmetic, assignment, comparison, logical, bitwise, string, conditional (ternary), type, and more — over 10 types in total.

Question 8: Explain Local Scope, Block Scope, Functional Scope, and Scope Chain in JavaScript

Local scope refers to variables declared within a function. Block scope is for variables inside curly braces (like those declared with `let` and `const`). Functional scope applies to `var` within functions. The scope chain is how JavaScript resolves variables by checking nested scopes outward.
